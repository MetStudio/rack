62 /*
63  Main features:
64 
65 
66  Cumulation. The compositing starts by initiating the target image,
67  see open().
68  Each input radar image is projected on the target image cumulatively,
69  see execute(). Finally, the cumulation must be completed by
70  normalizing the result with the number/weight of
71  cumulated inputs, see close().
72  Hence, the target which should be seen.
73 
74  The weights are stored in the first alphaChannel (index 0) of the
75  target image. If the target has a second image channel, also
76  the (weighted) squared values are cumulated.
77 
78  In cumulation, ...
79  \f[
80  x = \sqrt[p]{\frac{\sum_i {q_i}^r x_i^p}{\sum_i {q_i}^r } }
81  \f]
82 
83  The default type of target image, that is, cumulation array, is
84  double. If no (float valued) weighting is applied, target image
85  could be long int as well.
86 
87 \section specials Handling special codes undetect and nodata
88 
89 The implementation behind a composite is based on drain::Cumulator class, which essentially consists of four 2D arrays:
90 accumulated data (\f$\sum w_ix_i\f$), weights (\f$\sum w_i\f$), squared sum (\f$\sum w_ix_i^2\f$) and count of samples (\f$N\f$).
91 
92 For radar data ODIM standard defines two essential special codes, \c undetect and \c nodata, which above all mark samples that are
93 beyond the sensitivity or the range of a radar, respectively. \c nodata is also applied in marking samples where data has been rejected
94 due to noise and other anomalies. These codes imply adding special rules to the basically "mathematical" accumulation.
95 
96 In the accumulation stage, each valid data sample with data value \f$ x_i \f$ and its weight \f$ w_i \f$ is accumulated to the arrays by calling Cumulator::add(), with the following std::exceptions:
97 
98 - if \f$ x_i = \f$ \c undetect, just increment the count \f$N\f$ (which is equal to accumulating a sample with \f$ w=0 \f$)
99 - if \f$ x_i = \f$ \c nodata, skip the sample, leaving accumulation arrays intact
100 
101 In the extraction stage, a result with data value \f$ x_i \f$ and weight \f$ w_i \f$ are extracted at each data point according
102 to compositing principle (AVG, MAX,...) with the following special cases:
103 
104 - if \f$c_i = 0\f$ and \f$w_i = 0.0\f$, return result with \f$ x_i = \f$ \c nodata and \f$ w_i = 0.0 \f$
105 - if \f$c_i > 0\f$ and \f$w_i = 0.0\f$, return result with \f$ x_i = \f$ \c undetect and \f$ w_i = 0.0 \f$
106
