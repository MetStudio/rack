Detailed Description

template<class T, class P = drain::Path<std::string,'/'>, class C = std::less<std::string>>
class drain::Tree< T, P, C >

Unordered tree based on std::map.

Template Parameters
K- key type, implementing method empty() and cast from/to std::string.
T- value type
C- comparison functor, implementing less-than relation

tree[] returns childs only (complains separators, for some time) tree() returns descendants, allows deep paths.

Child nodes can be addressed with operator[] with std::string valued keys:

Tree<float> t;
t["firstChild"];

The keys can be retrieved to a list with getKeys().

Alternatively, one may iterate the children directly:

std::map<std::string,Tree<int> >::iterator it = tree.begin();
while (it != tree.end()){
std::cout << it->first << ' ' << it->second << '\n';
++it;
}

As the operator[] returns a reference, it may be used recursively:

t["firstChild"]["a"]["bb"];

Tree also recognizes a path separator, which is '/' by default. Hence, the node created in the above example could be likewise created or addressed as:

t("firstChild/a/bb")

Each node contains data of type T. It may be unused (empty), that is, a node can be used only as a branching point.

Tree<std::string> t;
t.data = "Hello world";
std::string s = t; // Now contains "Hello world".

(Direct assignments to and from the node data has been supported, but is now disabled for c++ compatibility reasons.)

Also direct referencing of the node data is now disabled:

Tree<std::string> t;
t = "Hello world";
Tree<std::string> &r = t; // Refers to tree.
std::string &s = t; // Refers to data, "Hello world".

Sorted tree structure.

Template Parameters
T- node type, containing user-designed data. Ideally, only this needed in template implementation.
P- applied path structure, essentially works like std::list
C- path element comparison functor, should be compatible with Path<P>::key_t.
